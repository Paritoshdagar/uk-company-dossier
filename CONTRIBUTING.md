# Contributing

Thank you for wanting to help. This project exists because practical public-register tooling gets better when practitioners, technologists, and curious users can point to real friction and improve the workflow together.

## Good ways to contribute

- Open an issue for a reproducible bug, confusing workflow, documentation gap, or useful Companies House use case.
- Share a clear example of the problem you are trying to solve, using public and non-sensitive information only.
- Open a focused pull request if you already have a practical fix, test, documentation improvement, or example.

## Pull request guidance

Pull requests are welcome, especially when they make the project easier to run, verify, understand, or adapt for non-commercial community use.

To protect contributors, users, and the project's non-commercial/dual-licensing model:

- submit only work you created or have the right to contribute;
- keep credentials, private company information, sensitive personal data, and confidential client material out of issues and pull requests;
- keep each pull request focused on one problem or use case;
- include enough context for reviewers to understand the issue, the solution, and how you tested it;
- expect maintainer review and, before merge, any contributor agreement or licensing confirmation needed for future dual licensing.

By opening a pull request, you agree that your contribution may be included in this repository under its current public licence. If a separate contributor agreement is needed before merge, the maintainer will say so clearly in the pull request.

If you are unsure whether something belongs in an issue or a pull request, start with an issue. A good problem statement is already a valuable contribution.
